1. FOUNDATION OF DATA STRUCTURES
--------------------------------------------------------------------------------
* Definition: Foundational elements and building blocks in programming used to 
  organize, store, and manage data efficiently within software applications 
  [cite: 2, 6].
* Importance: 
  - Essential for technical programming interviews[cite: 5].
  - Crucial for effective problem-solving, data science, and analytics[cite: 3, 7, 15].

  - Choosing data structures is like selecting materials (wood, bricks, glass) 
    to construct a house[cite: 5].

    selecting the wrong data structure breaks software efficiency[cite: 8].

    and manipulating information efficiently[cite: 9].
  - Arrays: Store data contiguously (side-by-side) in memory[cite: 12].
  - Dictionaries / Hash Maps: Store data using hash functions to enable fast, 
    direct key-based access rather than sequential index lookups[cite: 11, 12, 13].

2. BIG O NOTATION & COMPLEXITY ANALYSIS
--------------------------------------------------------------------------------
* Definition: A mathematical representation that measures how an algorithm's 
  running time or space requirements grow relative to the input size (n)[cite: 21, 25].
* Why Absolute Time Fails: 
  - Absolute time measurements are useless due to hardware and computer speed 
    differences[cite: 29, 30]. Big O provides a hardware-agnostic metric[cite: 30].
* Golden Rules of Big O Analysis:
  - Focuses strictly on the FASTEST GROWING TERM[cite: 31, 33].
  - Discards all constants and lower-order terms to simplify representation 
    (e.g., O(2n) becomes O(n))[cite: 31, 33].
  - Applied equally to both Time Complexity (running time) and Space Complexity 
    (memory footprints) [cite: 25, 40].
* Primary Complexity Benchmarks (Fastest to Slowest):
  1. O(1) - Constant Time: Running time remains almost identical regardless of 
     input size[cite: 35]. 
     - Example: Accessing an array element directly via its index[cite: 58].
  2. O(log n) - Logarithmic Time: Data/input size is systematically halved at 
     each step. Highly efficient for large datasets[cite: 41, 42].
     - Example: Binary Search[cite: 41, 42].
  3. O(n) - Linear Time: Running time increases proportionally with input size[cite: 27, 34].
     - Example: A single loop iterating through a dataset of size n[cite: 34, 59].
  4. O(n²) - Quadratic Time: Growth scales exponentially relative to the square 
     of the input size[cite: 36, 38]. 
     - Example: Nested loops or multi-block code structures[cite: 36, 37].

3. ARRAYS & DYNAMIC ARRAYS IN-DEPTH
--------------------------------------------------------------------------------
* Core Concept: A fundamental data structure used across languages to store data 
  sequentially[cite: 48, 51].
* Memory Mechanics:
  - Stored sequentially in RAM addresses using binary representation[cite: 55, 56].
  - In traditional lower-level compilation, each integer occupies 4 bytes of 
    memory[cite: 56].
* The Python Exception:
  - In Python, lists function as dynamic arrays[cite: 62].
  - Unlike Java/C++, Python stores object references internally, allowing a single 
    array to hold completely different data types simultaneously[cite: 61, 65].
* Static vs. Dynamic Array Architectures:
  - Static Arrays: Possess a rigid, fixed size declared at creation. Exceeding 
    the size boundary causes memory exceptions[cite: 63].
  - Dynamic Arrays: Grow automatically as needed[cite: 63]. They allocate an initial 
    memory buffer; when full, they double their capacity, incurring a one-time 
    overhead of copying old elements to the new block[cite: 64].

* Array Performance Profile (Big O Summary):
  +-------------------------+----------------+---------------------------------+
  | Operation               | Time Complexity| Contextual Behavior             |
  +-------------------------+----------------+---------------------------------+
  | Lookup / Access by Index| O(1)           | Calculated instantly via offset |
  | Search by Value         | O(n)           | Requires iterating/traversing   |
  | Full Traversal          | O(n)           | Iterates through all n elements |
  | Insertion (General)     | O(n)           | Requires shifting elements      |
  | Deletion (General)      | O(n)           | Requires shifting elements      |
  +-------------------------+----------------+---------------------------------+
  * Note: Shifting elements during insertion/deletion drives the O(n) cost[cite: 60].
